# ⚙️ راهنمای پیکربندی (Config)

<div dir="rtl">

ساده‌ترین (و توصیه‌شده‌ترین) راه راه‌اندازی، **منوی تعاملی** است: کافی است روی هر دو
سرور `skypass-tun` را اجرا کنی و گام‌به‌گام جلو بروی. لازم نیست هیچ فایل JSON‌ای دستی
بنویسی — سمت ایران یک **کدِ base64** می‌سازد و تو همان را روی سمت خارج paste می‌کنی.

بخش پایانی این سند («اجرای دستی از فایل JSON») مخصوص کاربران حرفه‌ای/اتوماسیون است.

---

## ترتیب درست راه‌اندازی

> 🔑 **اول لایسنس ← بعد ایران ← بعد خارج.**

```
┌── ۱. لایسنس ──────────────┐   ┌── ۲. سرور ایران ─────────┐   ┌── ۳. سرور خارج ──────────┐
│ پنل → صفحه‌ی Tunnel        │   │ skypass-tun → گزینه ۱     │   │ skypass-tun → گزینه ۲     │
│ IP سرور ایران را وارد کن  │──▶│ IP خارج + transport + پورت│──▶│ base64 را paste کن        │
│ → license key بگیر        │   │ → یک کدِ base64 می‌گیری    │   │ → license key را وارد کن  │
└───────────────────────────┘   └──────────────────────────┘   └──────────────────────────┘
                                         کدِ base64 ─────────────────────▶ paste
```

| سمت | کجا اجرا می‌شود | نقش | لایسنس؟ |
|-----|----------------|-----|---------|
| **ایران** (`iran`) | سرور داخل ایران | ورودی کاربران؛ **base64 را می‌سازد** | ❌ لازم نیست |
| **خارج** (`kharej`) | سرور خارج از ایران | خروجی؛ **base64 را paste می‌کند** | ✅ **اینجا وارد می‌شود** |

نکته‌ی مهم: `token`, `transport`, پورت‌ها و بقیه‌ی تنظیمات همگی داخل همان **base64**
هستند، پس روی سمت خارج نیازی به تنظیم دوباره‌شان نداری — فقط paste و سپس license.

---

## راه‌اندازی با منوی تعاملی (توصیه‌شده)

### روی سرور ایران — گزینه‌ی ۱ (Setup Iran side)

```
skypass-tun
# → 1) Setup Iran side
```

از تو می‌پرسد:

| پرسش | توضیح |
|------|-------|
| `tunnel name` | یک نام دلخواه (پیش‌فرض `iran1`) |
| `kharej server address (host:port)` | **IP و پورت سرور خارج**، مثل `1.2.3.4:8443` |
| `transport` | نوع حملِ داده (جدول پایین) |
| `ports` | پورت‌های forward، با کاما جدا شده — مثال زیر |
| `auto mode` | توصیه‌شده: `yes` (خودش tune می‌کند؛ نوعِ مصرف را می‌پرسد) |

فرمت پورت‌ها (هرکدام را با کاما جدا کن):

```
443,8080            → همان پورت روی 127.0.0.1
2525:2626           → listen 2525 به 127.0.0.1:2626
80:1.2.3.4:8080     → listen 80 به 1.2.3.4:8080
53/udp              → فقط UDP (بدون /tcp یا /udp = هر دو)
```

در پایان، خروجی چیزی شبیه این است — کدِ base64 را **کپی** کن:

```
════════ KHAREJ CONFIG (base64) ════════
paste this on the kharej server (menu option 2):

eyJuYW1lIjoiaXJhbjEiLCJzaWRlIjoia2hhcmVqIiwuLi4=
════════════════════════════════════════
```

### روی سرور خارج — گزینه‌ی ۲ (Setup Kharej side)

```
skypass-tun
# → 2) Setup Kharej side
# → paste base64 config:   ← کدِ مرحله‌ی قبل را اینجا paste کن
# → license key:           ← license را از پنل وارد کن
```

بعد از این، هر دو سمت تونل بالا می‌آیند. تمام.

> ⚠️ اگر license را خالی بگذاری، تونل با اخطار و **بدون enforcement لایسنس** اجرا
> می‌شود (توصیه نمی‌شود). license را از
> **[skypass.cloud/dashboard/tunnel](https://skypass.cloud/dashboard/tunnel)** بگیر.

## انتخاب transport

`transport` را روی **سرور ایران** انتخاب می‌کنی و داخل base64 به سمت خارج منتقل می‌شود.
مقادیر معتبر:

| مقدار | مناسبِ | توضیح کوتاه |
|-------|--------|-------------|
| `tcp` | عمومی، پایدار | ساده و مقاوم — اگر شک داری با این شروع کن |
| `tcpmux` | چند کاربر روی یک تونل | TCP + مالتی‌پلکس |
| `ws` | عبور از CDN/رِوِرس‌پروکسی | ترافیک شبیه وب‌سوکت |
| `wss` | مثل `ws` ولی رمزگذاری‌شده | امن‌تر روی مسیرهای حساس |
| `wsmux` / `wssmux` | چند کاربر روی یک تونلِ WS/WSS | مالتی‌پلکس |
| `quic` | تأخیر پایین (UDP) | مبتنی بر QUIC datagram |
| `quicmux` | QUIC چندکاربره | QUIC + مالتی‌پلکس استریم |
| `hysteria` | لینک‌های پرافت‌وخیز | QUIC + congestion تهاجمی + obfs |
| `gamepass` | **گیم / تماس تصویری** | datagram روی WebSocket (شبیه HTTPS)، بدون head-of-line |
| `ultimatepass` | **محیط‌های سخت‌گیر** | ترافیکِ کاملاً درهم‌ریخته + AEAD، ضدِ active-probe |
| `ultimategamepass` | **گیمِ کم‌تأخیر** | تونل KCP روی UDP — جبران سریع loss |
| `speedpassmux` | گیمِ چندکاربره | همان KCP + مالتی‌پلکس |
| `spoof` | سخت‌ترین فایروال‌ها | IP-spoofing روی UDP خام (فقط Linux + root) |

> اگر مطمئن نیستی با `tcp` شروع کن. برای بازی `ultimategamepass` یا `gamepass` را
> امتحان کن. برای دور زدنِ سخت‌گیرترین DPI سراغ `ultimatepass` برو.
>
> ⚠️ transport های UDP-محور (`quic`, `hysteria`, `ultimategamepass`, `speedpassmux`, `spoof`)
> نیاز دارند UDP بین دو سرور باز باشد.

## اجرای دستی از فایل JSON (پیشرفته)

اگر بخواهی به‌جای منو، تونل را از یک فایل `config.json` مستقیم اجرا کنی، ساختار
`TunnelConfig` این‌طور است. **فیلدهای الزامی:** `name`, `side`, `endpoint`,
`transport`, `token`, `ports`. بقیه اختیاری‌اند و مقدار پیش‌فرض دارند.

### نمونه — سمت ایران

```json
{
  "name": "my-tunnel",
  "side": "iran",
  "endpoint": "IP-سرور-خارج:8443",
  "transport": "tcp",
  "token": "یک-رمز-مشترک-قوی-بین-دو-سمت",
  "ports": [
    { "listen_port": 2087, "target": "127.0.0.1:2087", "tcp": true, "udp": false }
  ]
}
```

### نمونه — سمت خارج

همان config با `side: "kharej"`، `endpoint` روی حالت گوش‌دادن، و افزودنِ `license_key`:

```json
{
  "name": "my-tunnel",
  "side": "kharej",
  "endpoint": "0.0.0.0:8443",
  "transport": "tcp",
  "token": "یک-رمز-مشترک-قوی-بین-دو-سمت",
  "license_key": "SKP-XXXX-XXXX-XXXX-XXXX",
  "ports": [
    { "listen_port": 2087, "target": "127.0.0.1:2087", "tcp": true, "udp": false }
  ]
}
```

> 🔑 `token` و `transport` باید در **هر دو سمت یکسان** باشند؛ `license_key` فقط سمت
> خارج. (در حالت منوی تعاملی همه‌ی این‌ها خودکار از base64 هماهنگ می‌شوند.)

### توضیح فیلدها

| فیلد | الزامی | پیش‌فرض | توضیح |
|------|:------:|---------|-------|
| `name` | ✅ | — | نام دلخواه تونل (برای مدیریت چند تونل) |
| `side` | ✅ | — | `"iran"` یا `"kharej"` |
| `endpoint` | ✅ | — | ایران: `IP:PORT` سرور خارج · خارج: `0.0.0.0:PORT` برای گوش‌دادن |
| `transport` | ✅ | — | نوع حملِ داده (جدول بالا) |
| `token` | ✅ | — | رمز مشترک بین دو سمت |
| `ports` | ✅ | — | فهرست پورت‌های forward (پایین) |
| `license_key` | — | `""` | کلید license (**فقط سمت خارج**) |
| `sni` | — | `""` | SNI برای transport های TLS (`wss`/`quic`/…) |
| `mtu` | — | `1400` | MTU تونل؛ برای تونل‌تو‌تونل (WireGuard) کمتر امن‌تر است |
| `heartbeat_secs` | — | `30` | فاصله‌ی heartbeat (ثانیه) |
| `udp_timeout_secs` | — | `180` | timeout بی‌فعالیتی session UDP — برای بازی/WireGuard بالا نگه‌دار |
| `log_level` | — | `"info"` | سطح لاگ (`error`/`warn`/`info`/`debug`) |

### قانون پورت (`ports`)

هر عضوِ `ports` یک سرویس را از سمت ایران به سمت خارج می‌رساند:

| فیلد | توضیح |
|------|-------|
| `listen_port` | پورتی که سمت ایران روی آن گوش می‌دهد (کاربر به این وصل می‌شود) |
| `target` | مقصد در سمت خارج، مثل `127.0.0.1:2087` |
| `tcp` | آیا TCP تونل شود؟ (`true`/`false`) |
| `udp` | آیا UDP تونل شود؟ (`true`/`false`) |

چند سرویس هم‌زمان روی یک تونل:

```json
"ports": [
  { "listen_port": 2087,  "target": "127.0.0.1:2087",  "tcp": true,  "udp": false },
  { "listen_port": 8443,  "target": "127.0.0.1:8443",  "tcp": true,  "udp": false },
  { "listen_port": 51820, "target": "127.0.0.1:51820", "tcp": false, "udp": true  }
]
```

### دستورات CLI

```bash
# اجرای مستقیم از فایل JSON
skypass-tun run --config /etc/skypass/kharej.json

# گرفتن base64 سمت خارج از یک فایل JSON سمت ایران (معادل خروجی گزینه‌ی ۱)
skypass-tun export --config /etc/skypass/iran.json

# اجرا از رشته‌ی base64
skypass-tun run-b64 <BASE64>

# منوی تعاملی (ساده‌ترین راه)
skypass-tun
```

> 💡 `skypass-tun-update` تونل‌های در حال اجرا را خودکار روی نسخه‌ی جدید ری‌استارت
> می‌کند؛ نیازی به اجرای دستی `run` بعد از آپدیت نیست.
